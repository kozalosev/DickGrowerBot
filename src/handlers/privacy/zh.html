关于本机器人收集和存储哪些信息，最权威、最新且最完整的真相来源是 数据库迁移文件，它们和其余源代码一样发布在 GitHub 上。 不过，让我们简要地看一下其中最重要的部分： 1️⃣ 机器人必须存储用户的 ID 和名字，以便显示排行榜和 PvP 对战结果； 2️⃣ 需要用户账户的创建日期，以便提供欢迎奖励，即一段只增不减的宽限期； 3️⃣ 群组 ID 用于让用户在不同的群里拥有不同的丁丁，并支持机器人以指令模式和内联模式同时工作； 4️⃣ 显然，需要为每个群存储长度、最近一次增长的日期，以及贷款信息（金额、放款和还款日期、还款比例）； 5️⃣ 我们以累积表的形式收集胜负场次以及获得/失去厘米数的统计数据，而不会完整记录每一场对战的日志； 6️⃣ 以下情况会保留完整日志： ➖ 「今日丁丁」评选的获胜者（用于统计目的，以及判断今天是否已经选出过获胜者；不过这一点未来可能会被优化）； ➖ 从其他机器人导入的数据（用于排查问题，并防止指令被多次调用时重复导入）； ➖ 促销码的激活记录（用于统计和排查问题）。 7️⃣ 上述任何数据都不会提供给第三方机构，以下情况除外： ➖ 俄罗斯托管公司 TimeWeb，提供运行应用程序代码所需的云基础设施； ➖ 法国托管公司 Scaleway，提供兼容 S3 的存储服务用于保存数据库备份； ➖ 国际公司 Grafana Labs，其云基础设施中存储着各类指标和日志。
